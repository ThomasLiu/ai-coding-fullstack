## 1. 概述

### 1.1 目标

基于 OpenClaw + Claude Code 构建**真正的 AI Coding 全自动开发方案**，实现：

- 需求 → 设计 → 开发 → 测试 → 验收 → 部署 的全流程自动化
- 定时追踪 AI/AI Coding 领域最新动态
- 持续优化工作流，提效想法自动落地

### 1.2 核心原则

**Boil the Lake 原则**：当 AI 让边际成本接近零时，始终做完整的事。

## 3. 核心 Skill 设计

### 3.1 gstack-browse (浏览器自动化)

**参考**: gstack 的持久化浏览器 Daemon

**目标**: 解决浏览器自动化延迟问题

#### 技术方案

```
┌─────────────────────────────────────────────────────────────┐
│                  Playwright (持久化 CDP)                      │
├─────────────────────────────────────────────────────────────┤
│  首次启动: ~3s                                              │
│  后续调用: ~100-200ms/command                               │
│  Cookie/状态: 跨命令保持                                     │
└─────────────────────────────────────────────────────────────┘
```

#### Ref 系统

| Ref | 含义 | 示例 |
|-----|------|------|
| `@e1` | 第1个元素 | `<button>Submit</button>` |
| `@e2` | 第2个元素 | `<input name="email">` |
| `@c1` | 第1个内容块 | 文章段落 |
| `@b1` | 第1个按钮 | 提交按钮 |

#### 实现步骤

1. [ ] 集成 Playwright 到 Claude Code
2. [ ] 实现持久化 CDP 连接
3. [ ] 开发 ARIA Tree 生成
4. [ ] 实现 Locator 解析

### 3.2 qa (质量保证)

## 6. 与 gstack 的差异化

### 6.1 核心差异

| 维度 | gstack | 我们的方案 |
|------|--------|-----------|
| 调度器 | Claude Code | OpenClaw |
| 定时任务 | 无 | 有 (Cron) |
| 知识库 | 无 | 有 (kb-mini) |
| 浏览器自动化 | 原生 CDP | Agent Browser |
| Skills | 专业化角色 | 知识+行动 |

### 6.2 我们的优势

1. **OpenClaw 调度**: 原生支持 Cron、Hooks、多 Agent
2. **知识库**: 持续积累上下文
3. **定时追踪**: 自动采集最新信息
4. **每日反思**: 持续优化工作流
